# Panduan Kontribusi

Terima kasih telah tertarik untuk berkontribusi pada proyek K# Compiler! Panduan ini akan membantu Anda memahami cara berkontribusi.

## Cara Berkontribusi

1. **Fork repository** ini
2. **Buat branch** baru untuk fitur atau perbaikan yang ingin Anda tambahkan
3. **Lakukan perubahan** sesuai standar kode yang berlaku
4. **Tambahkan test** jika diperlukan
5. **Perbarui dokumentasi** jika diperlukan
6. **Buat pull request** dengan deskripsi yang jelas

## Standar Kode

- Gunakan format kode yang konsisten
- Tambahkan komentar untuk fungsi dan modul penting
- Ikuti struktur direktori yang telah ditentukan
- Pastikan kode dapat dibaca dan dimengerti

## Testing

- Pastikan semua test berjalan dengan baik sebelum membuat pull request
- Tambahkan test baru untuk fitur atau perbaikan yang Anda buat
- Gunakan `dune runtest` untuk menjalankan test suite

## Laporan Bug

Jika Anda menemukan bug, silakan buat issue dengan informasi berikut:
- Deskripsi bug yang jelas
- Langkah-langkah untuk mereproduksi bug
- Versi OCaml dan Dune yang digunakan
- Sistem operasi yang digunakan

## Permintaan Fitur

Jika Anda memiliki ide untuk fitur baru, silakan buat issue dengan:
- Deskripsi fitur yang jelas
- Alasan mengapa fitur tersebut dibutuhkan
- Contoh penggunaan jika memungkinkan

## Komunikasi

- Gunakan bahasa yang sopan dan profesional
- Diskusikan ide dan pendapat dengan terbuka
- Bantu sesama kontributor jika memungkinkan

## Lisensi

Dengan berkontribusi pada proyek ini, Anda setuju bahwa kontribusi Anda akan dilisensikan di bawah MIT License.
